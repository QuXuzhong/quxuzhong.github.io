MSCKF系列笔记：Hybrid VIO

Hybrid VIO是李明扬老师实现的EKF-SLAM和MSCKF融合的VIO方案。本文主要是阅读其原论文，梳理一下关键的思路和核心的公式。文章在开头指出，现有的VIO方案都是先设计出来，再看看能在什么算力的平台上去实时地运行。这种设计的策略问题在于既无法保证自己的结果在相同耗时下是最优的，也无法保证自己的结果在相同精度下是最快的。此外，不同的环境下不同的算法的表现也各有高低，因此作者旨在提出一种新的设计策略，即一个hybrid estimator。通过预先的统计信息，将计算耗时作为一个优化问题的代价函数来找到每个时刻下最优的方法。细化到具体的方法，本文选择了EKF-SLAM和MSCKF混合的方案。这两者在处理时哪一个更优取决于特征点的跟踪长度，在运行时会根据具体的场景解一个单变量的优化问题，决定到底用哪种方式更新。

内容概述

作者指出，EKF-SLAM和MSCKF使用的信息是完全相同的，只是公式的形式有所不同，如果观测模型是线性的，这两者理想的结果应该是完全一致的。所以将二者结合起来使用是一种非常自然的想法。二者最大的不同在于算力的耗时不用：EKF-SLAM每一步更新的耗时与特征点个数的三次方成正比，而MSCKF的计算耗时与特征点线性正比，却与特征点的跟踪时间的三次方成正比。由此可以得出，二者是天然互补的：MSCKF适合处理大量跟踪时间较短的特征点，而EKF-SLAM适合处理少量跟踪时间较长的特征点。

对于MSCKF系统，只有当前帧能跟踪到的所有特征点对应的共视帧都保留在滑窗内，即滑窗的长度长于被跟踪最久的特征点的跟踪帧数时，估计才是最优的（==我理解的最优就是指有效利用了所有的观测信息==）。但是在一般的情况下，能持续被跟踪的特征点只占很少一部分，文中列举的数据集上有88%的特征点只能被持续跟踪5帧以内，而被持续跟踪最久的特征点的被跟踪帧数可以达到40帧，此时如果将滑窗的长度设到40，计算会明显得低效和没有必要。针对这个问题，作者提出了EKF-SLAM的引入。

引入的策略很简单，跟踪帧数小于m的点会被用作MSCKF特征点（m为滑窗的长度），达到m次的点会被三角化、初始化其方差和协方差，加入到状态中作为EKF-SLAM的特征点。这些点三角化时采用其在最新帧的逆深度参数化方式表示。

整个流程可以分为update和state management两步：

Update：每当获取新的相机位姿就立刻执行

  • 增广状态向量
  • 对于MSCKF特征点（跟踪帧数少于m的点）：
    • 计算每个特征点的Jacobian和测量残差
    • 进行Mahalanobis gating test
    • 通过测试的特征点会被用来堆叠残差向量和Jacobian矩阵
  • 对已经被增广的EKF-SLAM特征点，计算Jacobian和观测残差
  • 进行EKF更新
  • 对跟踪长度达到m的特征点进行初始化

State Management：

  • 剔除掉不再被跟踪到的SLAM特征点，同时改变那些在最旧的帧下三角化的SLAM特征点的anchor pose(即改变其三角化的帧，因为最旧的帧马上要被剔除掉)
  • 从状态向量中剔除掉最旧的帧的位姿，检查一下是否比较旧的那些帧已经不再观测到当前追踪的任何特征点，将对应的帧都删掉（剔除帧的策略与MSCKF-VIO有些许不同）

Mahalanobis gating test

\[ \tilde{\mathbf{z}}_{i}^{o}=\mathbf{V}^{T} \tilde{\mathbf{z}}_{i} \simeq \mathbf{V}^{T} \mathbf{H}_{i} \tilde{\mathbf{x}}+\mathbf{V}^{T} \mathbf{n}_{i}=\mathbf{H}_{i}^{o} \tilde{\mathbf{x}}+\mathbf{n}_{i}^{o} \]

算法复杂度分析和最优策略

将MSCKF和EKF-SLAM混合后，得到第k次更新时总的计算复杂度为： \[ \begin{aligned} f_{k}=& \alpha_{1} \sum_{i=1}^{n} \ell_{i}^{3}+\alpha_{2}\left(r+2 s_{k}\right)^{3} \\ &+\alpha_{3}\left(r+2 s_{k}\right)\left(15+6 m+3 s_{k}\right)^{2}+\text { l.o.t } \end{aligned} \] 其中\(\alpha_i\)都是常数==？？？==，\(l_i\)对应第i个特征点被跟踪的帧数，r表示所有的MSCKF特征点对应的相机位姿的自由度，作者表示其上限是6m-7，减掉的7个自由度是相对世界坐标系的位姿和尺度==？？？==。\(s_k\)是增广的SLAM特征点数。l.o.t表示其余的低阶项，忽略掉。如果取r=6m-7，这个方程由m和\(s_k\)确定，但是注意到\(s_k\)事实上是由m决定的，因为它表示的是满足被跟踪长度不小于m的特征点。所以计算复杂度实际上由m唯一确定，于是问题就变成了如何求解最优的m。

但是这个问题没有办法以在每一时刻求解最优的m的方式进行解决，道理很简单，如果在当前的更新步骤时窗口长度为20，且有10个特征点观测次数刚好达到了20，我们并没有办法确定到底是该将这10个点扩增到状态中还是将窗口扩大到21，如果这些点未来的被跟踪长度远远大于20，显然扩增状态是更合适的，而如果这些点只能被跟踪21帧，扩大窗口就更合适。

然后作者提出的解决方案我就看不懂了，貌似是通过大量的数据统计了每个特征点被跟踪长度的pmf(probability mass function，概率质量函数)，然后利用这个去算最低的平均EKF更新时间复杂度\(\overline f(m)\)，找到最小化的值m。实验表明这个函数是个凸函数，可以找到唯一的极小值。

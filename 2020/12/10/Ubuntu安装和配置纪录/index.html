Ubuntu安装和配置纪录

新硬盘到了，决定重新装下win10，再整个ubuntu，又要重新装系统配环境，决定写个大致的步骤留个备份。相关的教程网上很多了，win10重装不会有太大坑，大部分教程都是对的，ubuntu问题多些。主要记一下遇到的坑。

系统安装

1.电脑：rog魔霸新锐 ubuntu版本:18.04.05 留了256G用来存系统，用Ultrso往u盘里写系统，不必多说

2.开机按f2进bios，把SATA模式改成AHCI，advanced 选项里把security boot关掉（这两项不修改的话装系统的时候识别不到硬盘），把u盘调到启动选项的第一个。

3.开机，装系统，安装方式的部分选最后的其他，分了四个区

挂载点 / /boot /home swap
空间大小 60G 2G 剩下的200G 16G
格式 Ext4 Ext4 Ext4 交换空间
类型 主要分区 主要分区 逻辑分区 主要分区
作用 相当于C盘 系统文件 主要工作目录 跟电脑内存一样大

4.傻瓜式装机结束后，进BIOS把ubuntu调到启动顺序的第一个，开始被整蛊。

  • windows有BitLock的情况下由于重装系统关掉了secured boot，需要输秘钥，原理上讲只需要登陆自己的Microsoft账号查看秘钥就可以，但是我一直没成功。方法参见此博客
  • 于是把步骤2修改的选项改回去，主要是把AHCI改回去，再把windows系统启动顺序放到第一个，进入windows系统以后再把BitLocker都关掉。关的时候需要登陆Microsoft账号
  • 然后改回AHCI还是不能进windows系统，折腾了很久发现最好是在AHCI模式下去装windows系统。。。用easyBCD搞了半天把Windows引导搞没了，一怒之下直接格式化了所有硬盘，从头再来。

==正确操作==

开AHCI->装windows，关掉BitLock->装ubuntu，选boot为引导盘，搞定，终于搞定了:anger::fu:。开机装软件！

装东西之前先搞一下双屏显示，万万没想到这一步也僵住了，在display选项里是可以很方便地设置双屏显示的分辨率、位置之类的，但是第二个屏幕只亮了个壁纸，拖东西过去的话窗口都神秘消失了，只看得到鼠标在无力地挥来挥去​，人间迷惑。参考了网上很多双屏显示的博客，很多都写得非常复杂，有点不敢下手，毕竟已经重装了两次，顶不住了。但总之得出个结论，应该是跟显卡驱动的设置有关。于是打开software&updates设置，找到additional drivers，默认的话选中的是Nouveau display driver，这个驱动貌似双屏输出会有问题，同时这个设置界面提供了很多N卡驱动，直接安装了最新的那个，重启。

解决了！

开机弹错误的问题

开机一直弹出System program problem detected，是因为系统遇到过崩溃的情况，其实对使用没什么影响，但是强迫症被搞得很不爽，关掉通知的方法：打开/etc/default/apport，将其中的enable=1改为0即可。

软件安装

xkill:先搞个快捷键，防止窗口无响应。keyboard选项里拉到最后，添加，指令就是xkill,快捷键自己设（我用了ctrl+alt+x)

截屏快捷键：同样在keyboard里添加，指令填入gnome-screenshot -a，设定快捷键即可。截取的图片会默认保存在pictures目录下

输入法

装系统为了以后使用方便设置了英文，所以要调一下中文输入法，使用系统自带的中文输入法即可，在Language选项选择install，安装Chinese-simplified，之后打开Settings 里面的Region &Language选项，添加Input Sources，选Chinese以后会进一步有Chinese(Intelligent Pinyin)选项，添加好就ok。如果没有Intelligent Pinyin选项的话是Language选项的install没成功。添加好以后在右上角可以切输入法了，记得把记忆输入打开，把词库里面各种类型都勾了。建议最好不要用搜狗输入法，容易跟各种编译器起冲突。自带的编译器虽然开始用的时候词条的联想不太行，但开了记忆输入会越用越顺手的。

typora

轻量的markdown笔记本，非常好用，装好系统想记录一下，先下载Typora，直接在官网选linux系统，会有详细的安装指令。装好以后在File-Prefrence里面把各种markdown选项都勾了，把图片存储路径改到当前文件夹，View选项里面打开打字机模式，搞定。

electron-酸酸乳

chrome用习惯了，装一哈。装好要同步插件，需要富强民主文明和谐，ubuntu一直用elctron，按照github里面的说明把需要装的都装了，然后打开Network选项，把Proxy设置成手动，HTTP、HTTPS、Socks都改成127.0.0.1 +12333，搞定。

stretchly

最近眼睛不是很舒服，为了保证自己为祖国健康工作五十年，装一下休息提示软件，直接在软件的github源找对应的deb包安装即可。

vlc

比较常用的视频播放器，安装:sudo apt install vlc

goldendict

划词翻译的软件。安装:sudo apt-get install goldendict，之后可以配置在线辞典，Edit->Dictionaries选择website，有道辞典链接为：http://dict.youdao.com/search?q=%GDWORD%&ue=utf8

C++开发环境

基本环境

gcc和g++（编译工具）: sudo apt-get install gcc(g++)

cmake: sudo apt-get install cmake

clion:比较好用的跨平台IDE，直接官网下载deb包即可，可以用教育邮箱进行认证。

  • 这IDE很好用，但是可能出现IDE内不能输入中文的BUG，解决方案：

    去JBR的官网下载一个跟自己CLION发行时间差不多的版本，然后直接在Clion的文件夹内把旧的JRB文件替换掉，下载地址

opencv

这个不用多说吧。犹豫了下要不要试试新鲜出炉的4.5版本，想到要跑的一些旧项目，还是乖乖装了opencv 3.4.12。自己写了几个脚本，安装步骤如下（安装之前请先查看下面遇到的问题，确定你有没有对应的需要）：

  1. 下载opencv和对应版本contrib的deb包，github官方地址：link
  2. 先安装需要的其他库，运行脚本opencv_dependencies.sh即可
  3. 在opencv目录下运行install.sh即可

安装遇到的问题：

~/opencv_contrib/modules/xfeatures2d/src/boostdesc.cpp:673:20: fatal error: boostdesc_bgm.i: No such file or directory

比较麻烦的方法是查看build文件夹里面的CMAKEDownloadLog，搜索缺少的文件（比如boostdesc_bgm.i），会找到此文件的网址，打开是文件的源码，只需要在xfeature2d的src目录新建此文件，把源码拷进去就可以。但是缺少的文件很多，一个个下比较麻烦。还好有博主放了全部文件的镜像，下载。全部复制到xfeature2d的src目录下就搞定了。

安装以后查看版本：pkg-config --modversion opencv

使用LSD

比较新的opencv版本移除了对LSD算法的支持，为了使用lsd方法，用opencv3.4.3中modules/imgproc/src/lsd.cpp换掉了3.4.12中的对应文件，然后重新编译安装，解决了问题。

使用SURF

比较新的版本安装额外库并不能使用SURF，需要开启non-free模块，找到opencv目录下CmakeLists，有对应选项：

OCV_OPTION(OPENCV_ENABLE_NONFREE "Enable non-free algorithms" ON)

本来是OFF，改成ON，按照上面的步骤安装即可，如果还是不行，可以尝试在install.sh里cmake选项中加上-D OPENCV_ENABLE_NONFREE=ON \，但是我这么干出Bug了，其他版本的opencv也许可以

eigen

矩阵计算库，只包含头文件，所以不需要编译，下载在工程里include即可，如果要使用find package来检索，需要创建软链接。安装步骤：

  • 安装：sudo apt-get install libeigen3-dev

  • 进入目录检查eigen3文件夹位置，通常会安装在usr/include，也可能在usr/local/include，这个不是很明确原因。

  • 在该目录创建软链接，类似于为文件创建一个快捷方式

    sudo ln -sf eigen3/Eigen Eigen

    sudo ln -sf eigen3/unsupported unsupported

  • 使用只需要在cmake里直接find_package(Eigen3 REQUIRED)

blas

一个基础性线性代数库。

  • 安装：sudo apt-get install libblas-dev

  • 使用：要链接的只有一个动态库文件，即libblas.so.3，如果在usr/lib中查看有两个地方有这个文件，按下面的路径链接

    SET(BLAS_LIB /usr/lib/x86_64-linux-gnu/blas/libblas.so.3) TARGET_LINK_LIBRARIES(LineMatchingLib ${BLAS_LIB} )

SuperLu

一个求解器。

  • 安装：sudo apt-get install libsuperlu-dev
  • 使用：参照blas库，在同一个文件路径里。

arpack++

一个用来求解稀疏矩阵的库，arpack的高级版，使用需要先安装arpack

  • 安装：sudo apt-get install libarpack-2dev

    ​ sudo apt-get install libarpack++2-dev

  • 使用：

    SET(ARPACK_INCLUDE_DIR "/usr/include/arpack++")，告诉工程头文件的所在路径

    SET(ARPACK_LINK_DIR "/usr/lib"),告诉工程库文件即.a,.so文件所在路径

    INCLUDE_DIRECTORIES(${ARPACK_INCLUDE_DIR})，引用头文件

    TARGET_LINK_LIBRARIES(LineMatchingLib arpack arpack++ )，注意这里把两个库都链接

Sophus

一个李群李代数的库，比较新的是模板类的版本。

  • 安装：link，下载后cmake，make即可，如果使用非模板类，需要在Sophus目录下使用git checkout a621ff

  • 使用：

CXSparse

需要换源安装

  • 换源：sudo gedit /etc/apt/sources.list

    添加：deb http://cz.archive.ubuntu.com/ubuntu trusty main universe

    更新：sudo apt-get update

g2o

图优化库

  • 安装：[link]，按照页面内的指引即可，但为了使用方便在make之后可以安装，执行sudo make install即可

  • 使用：g2o的find模块需要手动添加，将**替换为下载g2o的路径

ceres

手册比较全的开源优化库

  • 安装：link，去官网查看需要的依赖项即可
  • 使用：find_package(Ceres REQUIRED)

python开发环境

python3和python2.7的指令基本一样，只需要将python改称python3即可，本文以python3为例

安装python:sudo apt-get install python3-dev

pip是Python包管理工具，安装命令：sudo apt install python3-pip

常用扩展包:

  • numpy:sudo apt-get install python3-numpy或者pip3 install --user numpy，--user是安装到用户目录下
  • scipy:sudo apt-get install python3-scipy
  • matplotlib:sudo apt-get install python3-matplotlib
